將數字轉二進位後，計算從a寫到b需要幾個1  
ex: 寫出5到10需要12個1  
![image](https://user-images.githubusercontent.com/66452317/158021750-2820ff16-58f8-42e7-bced-f925dad44f77.png)

### 解法
因為數字的範圍很大，所以不能用暴力解為每個數建表，必須個別case計算(但還是要先建個小表)   
首先我們知道每進一位，後面的數字都會開始做輪迴  
所以可以分成算開頭的1的個數 + 後面輪迴的個數  

假設現在在算第k位數的1總共有多少   
後面的數字輪迴總共會做2^(k-1)次，所以計算開頭為1的次數 = 2^(k-1)，後面的數字開始重複之前的排列 + table[k-1]

#### table[k] = table[k-1] + [ 2^(k-1) + table[k-1] ]
####          = 2^(k-1) + table[k-1]*2
```
  0
  1 2^(0) + 0*2 = 1
 10 
 11 2^(1) + 1*2 = 4
100
101  
110
111 2^(2) + 4*2 = 12
1111 2^(3) + 12*2 = 32
```
#### 計算的方法
由table[k]去減  
如果bit 為 0 就表示少掉了開的1與後面的輪迴  
```
100
1 -> 12
0 -> -2^1(開頭的1) - (4-1)(後面輪迴的1) = -5
0 ->  -2^0 - (1-0)  = -2
12-5-2=5
```

```
1010
1 -> 32
0 -> -2^2 - (12-4) = -12
1 ->  0  (-2^0) 如果之後為0的話就要減
0 -> -2^0 - (1-0)  = -2
32-12-2=18
```
ans = 18 - 7 + 1
